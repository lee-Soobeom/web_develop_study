# html_study

### 태그 사이 관계

- 선조: 기준 태그의 한단계 이상 상위 태그를 의마한다
- 부모: 기준태그의 한 간계 상위 태그를 의마한다
- 자식: 기준 태그의 한 단계 하위 태그를 의미한다.
- 손자: 기준 태그의 한 단계 이상 하이 태그를 의미한다.
- 형제: 기준태그와 동일한 부모를 가지고 보다 후행하는 태그를 의미한다
- 인접 형제: 기준태그와 동일한 부모를 가지고, 바로 뒤에 후행하는 태그를 의미한다

## 속성

- HTML 태그는 속성을 가질수 있다.
- 속성은 여는 태그에 속성 이름, 등호, 쌍따옴표를 활용하여 `<태그 속성="값">` 의 형식으로 작성할수있다.
- HTML언어에서 속성값으로 작성하는 모든 값은 문자열임에 유의한다.
- `속성="값"` 형태로 여러개의 속성을 부여할 수 있다.
- 단순히 속성 이름만 적어 값이 없는 속성을 작성할 수 있다.
- 동일한 이름을 가지는 속성을 여러개 부여하지 않는다.
- 속성이 값을 가지지 않는다하여 의미가 없는것은 아니며, 일부 속성은 단순히 정의되어 있는 것 만으로도 의미가 부여되는 경우가 있다.

## `<a>` tag
- a (Anchor) 태그는 현재 페이지에서 다른 페이지로 이동할 수 있는 링크를 만들기 위해 사용한다.
- 스스로 닫는 태그 아니다.
- 속성
	- href : 이동할 주소 및 경로
 	- target : 이동할 방식
  		- _blank : 새로운탭
    	-  _self : 현재탭

`<a href="" target="_blank"></a>`

`<a href="" target="_self"></a>`

## `<b>` tag
- b 태그는 텍스트를 굵게(Bold) 만든다.
- 스스로닫는 태그가 아니다.
- 해당 태그가 가지는 내용이 문맥상 중요하다는 의미를 가지지만, 이러한 경우 주로 strong이나 em 태그를 사용하는 것이 좋다.
- 구시대에는 단순히 글씨를 굵게 하기 위해 사용하였으나, 현대 프론트에서는 단순히 글씨를 굵게하기 위한 용도로의 사용을 권하지 않는다. 대신 css의 font-weight 속성을 사용하여야한다.

## `<strong>` tag
- strong 태그는 텍스트를 굵게 만든다.
- 스스로 닫는 태그가 아니다.
- 문서 내에서 중요한 문장이나 단어를 포함하고자 할때 사용한다. 단, 문서, 문단의 제목이나 문단 전체를 포함하여서는 안된다.
- (짧은 길이 강조)

## `<cite>` tag
- cite 태그는 글씨를 기울인다.
- 스스로 닫는 태그가 아니다.
- 출판물, 영화 등의 저작물이나 출처, 제목 등을 나타내기 위해 사용한다.

## `<em>` tag
- em (Emphasize) 태그는 글씨를 기울인다.
- 스스로 닫는 태그가 아니다.
- 문서 내에서 중요한 문장이나 단어를 포함하고자 할 때 사용한다. 단, 문서, 문단의 제목이나 문단 전체를 포함하여서는 안 된다.

## `<i>` tag
- i (Italic) 태그는 글씨를 기울인다.
- 스스로 닫는 태그가 아니다.
- 본문의 주된 언어와 다른 언어를 담기 위해 사용한다.(핵심)
- 단순히 글씨를 기울이기 위해 사용하여서는 안되며, 이를 위해서는 css의 font-style 속성을 사용하여야 한다.

## `<br>` tag
- br (Break) 태그는 개행을 위해 사용한다.
- 스스로 닫는 태그다.
- 문단을 나누거나, 레이아웃(디자인) 목적으로 사용하여서는 안된다.

## `<p>` tag
- p (Paragraph) 태그는 문단 하나를 담기 위해 사용한다.
- 스스로 닫는 태그가 아니다.

## `<img>` tag
- img (Image) 태그는 이미지를 삽입하기 위해 사용한다.
- 스스로 닫는 태그다.
- 속성
	- alt : 명시한 이미지를 표시할 수 없을 때 대신(Alternative)하여 표시할 텍스트이다. 이미지에 대한 간단한 설명이 되기고 하고, 시각 장애인의 접근성을 크게 향상시킨다.
	- src : 표시할 이미지의 주소, 경로 혹은 64진법 문자열 데이터이다.
 
`<img alt="" src="">`

## h1~h6 tag
- h1 h2 h3 h4 h5 h6 Heading 태그는 이를 포함하는(컨테이너) 태그의 제목이나 간략한 주제를 나타내기 위해 사용한다.
- 스스로 닫는 태그가 아니다.
- 숫자가 작을수록(1에 가까울수록) 의미가 큰 태그이다.
- 단, h1 태그는 문서의 제목과 같아야 하고, 문서에 단 하나만 존재해야 한다.
```
    <h1>라면은 맛있다</h1>
        <h2>풍부한 맛의 조화</h2>
            <div>
                <h3>감칠맛</h3>
                <p>...yummy...</p>
            </div>
            <div>
                <h3>매운맛</h3>
                <p>...nice...</p>
            </div>
```

## `<li>` tag
- li (List Item) 태그는 목록 태그(ol, ul)의 항목 하나를 가지기 위한 태그이다.
- 스스로 닫는 태그가 아니다.
- 부모로 ol 혹은 li 태그를 가진다.


## `<ol>` tag
- ol (Ordered List) 태그는 순서와 관련이 있는, 서로 관련된 항목을 가지기 위한 목록 태그이다.
- 스스로 닫는 태그가 아니다.
- 자식으로 li 태그를 가진다.

## `<ul>` tag
- ul (Unordered List) 태그는 순서와 관련이 없는, 서로 관련된 항목을 가지기 위한 목록 태그이다.
- 스스로 닫는 태그가 아니다.
- 자식으로 li 태그를 가진다.

## `<sub>` tag
-sub (Subscript) 태그는 아래 첨자를위해 사용하는 태그이다.
- 스스로 닫는 태그가 아니다.

## `<sup>` tag
- sup (SuperScript) 태그는 위 첨자를 위해 사용하는 태그이다.
- 스스로 닫는 태그가 아니다.

## `<code>` tag
- code 태그는 간단한 코드 조각을 나타내기 위해 사용한다.
- 스스로 닫는 태그가 아니다.

## `<pre>` tag
- pre (Preformatted) 태그는 HTML 문서상에 작성한 내용 그대로 표현하기 위해 사용한다.
- 스스로 닫는 태그가 아니다.
- 들여쓰기 및 연속 공백을 모두 인정한다.

## table
table 태그는 표를 만들기 위한 태그이다.
스스로 닫는 태그가 아니다.
데이터를 담고 시각화하기 위한 목적으로만 사용하여야 하며, 사이트의 레이아웃이나 디자인 목적으로 사용하여서는 안된다.

## thead
 thead (Table Head) 태그는 테이블이 데이터에 대한 열 (Column)을가지기 위해 사용한다.
스스로 닫는 태그가 아니다.
실질적인 데이터를 담지 않는 것이 좋다.
table 태그의 자식이어야 하며 한 개 이하만 사용되어야 한다.

## tbody
tbody (Table Body) 태그는 테이블의 실질적인 데이터를 담기위해 사용한다.
스스로 닫는 태그가 아니다.
table 태그의 자식이어야 하며 한 개 이상이 사용되어야 한다.

## tfoot
tfoot (Table Foot) 태그는 테이블이 가지고 있는 데이터의 통계나 정리, 필터링을 목적으로 사용하는 경우가 많다.
스스로 닫는 태그가 아니다.
table 태그의 자식이어야 하며 한 개 이하만 사용되어야 한다.

## tr
tr (Table Row) 태그는 행 하나를 정의하기 위해 사용한다.
스스로 닫는 태그가 아니다.
행의 존재를 정의할 뿐, 데이터를 직접 가질 수는 없다.
thead, tbody, tfoot 의 자식이어야 한다.

## td
td (Table Data Cell) 태그는 셀(칸)을 정의하는데 사용한다.
스스로 닫는 태그가 아니다.
실제 데이터를 담는 목적으로 사용한다.
tr 태그의 자식이어야 한다.
-속성
colspan="" : 차지할 열(가로) 방향으로의 칸 개수를 지정한다. 개수에는 속성이 부여되는 칸도 포함된다. 즉, 값이 1이라면 확장하지 않는다.
rowspan="": 차지할 행(세로) 방향으로의 칸 개수를 지정한다. 개수에는 속성이 부여되는 칸도 포함된다. 즉, 값이 1이라면 확장하지 않는다.

## th
th (Table Heading Cell) 태그는 셀(칸)을 정의하는데 사용한다.
스스로 닫는 태그가 아니다.
열이나 행을 대표하는 이름이나 설명을 담기 위해 사용하며, 실제 데이터를 담아서는 안 된다.
-속성
colspan="" : 차지할 열(가로) 방향으로의 칸 개수를 지정한다. 개수에는 속성이 부여되는 칸도 포함된다. 즉, 값이 1이라면 확장하지 않는다.
rowspan="": 차지할 행(세로) 방향으로의 칸 개수를 지정한다. 개수에는 속성이 부여되는 칸도 포함된다. 즉, 값이 1이라면 확장하지 않는다.
scope="": 어느 방향의 데이터에 대한 대표 값인지에 대한 설정. 별도로 명시하지 않으면 브라우저가 스스로 판단한다.
-col: 열대표(세로 방향)
-row: 행 대표(가로 방향)

##form
form (양식) 태그는 서버로 전송하고자 하는 사용자 입력 값을 받는 입력 태그들(input, select, textarea 태그 들)을 자식 혹은 자손으로 가지기 위한 태그이다.
스스로 닫는 태그가 아니다.
하나의 양식 태그는 서로 유관한 입력 태그들만 가지는 것이 좋다.
양식 태그는 서로 선조, 부모, 자식, 자손 관계를 형성하여서는 안된다.
-속성
enctype: 데이터를 전송할때 인코딩할 방식을 지정한다.
	-application/x-www-form-urlencoded: 문자열 데이터를 URL 인코딩 방식으로 인코딩하여 전송한다. 별도의 명시가 없다면 기본값.
	-*multipart/form-data: 전송하는 데이터를 여러 조각으로 나누어 전송한다. 주로 첨부 파일 업로드와 같이 큰 데이터를 전송할때 사용한다.
	-text/plain: 문자열 데이터를 그대로 전송하되 공백만 + 기호로 치환한다.
method: 요청을 전송할 방식을 지정한다.
	-get: GET 방식으로 지정 (url)
	-post: POST 방식으로 지정 (login)
name: 양식 태그의 이름을 지정한다. (자바스크립트에서 document.forms 멤버의 키 값으로 접근 가능)
novalidate: 양식 태그의 제출(submit) 이벤트가 발생하였을때 이가 가지고 있는 입력 태그들이 가지고있는 요구사항을 무시하게 한다. 속성 값을 가지지 않음.

##label
label 태그는 입력 태그와 관련하여 이에 대한 설명을 작성하고 연결시키기 위해 사용한다.
스스로 닫는 태그가 아니다.
label 태그는 연결된 입력 태그에 대한 설명을 문자열로 가지며, 스크린 리더가 연결된 태그에 대한 설명으로 해당 내용을 사용한다.
특별한 경우가 아니라면 입력 태그를 하나만 가지도록 작성하여야한다.
label 태그 사용하는 방법은 아래와 같다
	-label 태그가 직접 입력 태그를 자식 혹은 자손으로 가지게 하느 방법.
	<label>
		<span>email</span>
		<input name="email" type="email">
	</label>
	-label 태그의 for 속성 값을 입력 태그가 가지는 id 속성 값과 일치 시키는 방법
	<label for="email">email</label>
	<input id="email" name="email" type="email">
	id 속성 값은 하나의 HTML 문서 내에서 고유해야함으로 각별히 유의한다

##input
input 태그는 사용자로부터 입력값을 받기 위해사용하는 입력 태그이다.
스스로 닫는태그이다.
-속성
name: 값을 서버로 전송할때 사용할 식별자이다.
value: 해당 입력 태그의 (초기)값을 지정한다.
type: 해당 입력 태그의 종류를 지정한다.
	button: 별다른 특성이 없는 일반 버튼. 표시할 테스트는 value 속성 값으로 지정한다.
	checkbox: 논리값(참, 거짓)
	color: 색상. 값은 HEX 컬러인 문자열로 전송된다.
	date: 날짜.
	datetime-local: 날짜와 시간.
	email: 이메일.
	file: 파일.
	-양식 태그의 enctype 속성 값을 multipart/form-data로 설정한다.
	-양식 태그의 method 속성 값을 post로 설정한다.
	hidden: 숨겨진 입력 태그를 위해 사용한다. 주로, 서버로 전송은 되어어야 하나 사용자가 직접 입력하는 값이 아닌(보안상의 이유로 대조되는 값 등) 것들을 포함하기 위해 사용한다.
	number: 숫자.
	password: 비밀번호. (마스킹***되는 텍스트)
	radio: 여러 선택지 중 하나를 선택할 수 있게 하기 위해 사용한다. name 속성 값이 같은 radio 중 하나만 선택 할 수 있다.
	range: 범위 내위 숫자를 선택하기 위해 사용하는 막대.
	reset: 속해있는 양식 태그가 가지고 있는 입력 태그들의 값을 초기화 하는 버튼. 표시할 텍스트는 value 속성 값으로 지정한다.
	search: 검색 키워드를 입력받기 위해 사용한다.
	submit: 속해있는 양식 태그를 제출하기 위함 버튼. 표시할 텍스트는 value 속성 값으로 지정한다.
	tel: 연락처.
	text: 별다른 특성이 없는 일반 텍스트.
	time: 시간.
accept: type 속성값이 file 일때 파일 선택 창에서 기본적으로 선택하게 할 파일의 형식을 지정한다. 별도로 지정하지 않으면 모든 파일을 선택할 수 있게 한다.
	.확장자(가령 .jpg 등)
	MIME(가령 image/jpg 등)
	MIME 타입의 서브타입은 와일드카드를 허용한다. (가령 image.*등)
	위와 같은 형식을 콤마(,)로 구분하여 열개 작성할 수 있다. (가령 .jpg, .jpeg, .png, .gif)
autofocus: 페이지가 로드되었을때 해당 입력 태그에 자동으로 포커스를 줄 것인가에 대한 속성. 속성 값을 가지지 않음.
checked: type 속성 값이 checkbox 이거나 radio 일 때, 체크된 상태로 설정할 것인가에 대한 속성. 속성 값을 가지지 않음.
disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않음.
max: type 속성 값이 number, range 등 숫자와 관련이 있을 때 가질 수 있는 최댓값.
min: type 속성 값이 number, range 등 숫자와 관련이 있을 때 가질 수 있는 최솟값.
maxlength: 문자열의 최대 길이. (값이 비어있을 때 검사하지 않음)
minlength: 문자열의 최소 길이. (값이 비어있을 때 검사하지 않음)
multiple: type 속성 값이 file 일때 파일 여러개를 선택할 수 있게 한다. 속성 값을 가지지 않음.
placeholder: 값이 비어있을 때 보여줄 텍스트를 지정한다.
readonly: 읽기 전용으로 지정하기 위해 사용한다. 속성 값을 가지지 않음.
required: 필수 필드이며 값이 비버있을수 없음을 의미한다. 속성 값을 가지지 않음.
spellcheck: 브라우저에 내장되어있는 철자 확인 기능을 사용할 것인가에 대한 설정.
    default: 기본값으로, 브라우저에 설정에 따름
    true: 사용
    false: 미사용
step: type 속성 갓ㅂ이 number, range 등 숫자와 관련이 있을 때 변화하는 단위를 설정한다.

##select
select 태그는 드롭다운(Drop Down) 방식 혹은 리스트(List) 방식의 할목 선택을 위한 입력 태그이다.
스스로 닫는 태그가 아니다.
select 태그는 직접 값을 가지지 않으며, 자식 혹은 자손으로 값을 가지는 option 태그를 가진다.
속성
name: 값을 서버로 전송할 때 사용할 식별자이다.
autofocus: 페이지가 로드되었을 때 해당 입력 태그에 자동으로 포커스를 줄 것인가에 대한 속성. 속성 값을 가지지 않음.
disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않음.
multiple: 여러개의 option 을 선택할 수 있게 한다. 드롭 다운혈식에서 리스트 형식으로 변하게 된다. 속성 값을 가지지 않음.
required: 필수 필드이며 값이 비어있을 수 없음을 의미한다. 속성 값을 가지지 않음.

##optgroup
optgroup (Option Group) 태그는 서로 유관한 option 태그들을 묶어내기 위해 사용한다.
스스로 닫느 태그가 아니다.
하나 이상의 option 태그가 서로 관련이 있음을 알리지만, 선택이 불가한 항목이고, 시각적 외의 특별한 기능을 가지지 않는다.
select 태그의 자식이어야 하고, option 태그를 자식으로 가져야 한다.
속성
label: 표시할 텍스트를 지정한다.
disabled: 해당 태그를 비활성화 한다. 단, 이가 가지고 있는 option 태그들도 함께 비활성화 된다. 속성 값을 가지지 않음.

##option
option 태그는 select 태그가 가지는 실질적인 값을 가지는 항목 태그이다.
스스로 닫는 태그가 아니다.
속성
disabled: 해당 선택지를 비활성화 한다. 속성 값을 가지지 않음.
hidden: 해당 선탣지를 숨긴다. 속성 값을 가지지 않음.
selected: 해당 선택지를 초기에 선택된 상태로 지정한다. 속성 값을 가지지 않음.
value: 실질적으로 서버에 전송될 값을 지정한다. 생략할 경우 해당 선택지가 가지고 있는 내용으로 대체된다.

##button

##textarea
textarea 태그는 개행이 가능한 문자열 값을 입력받기 위해 사용한다.
스스로 닫는 태그가 아니다.
태그의 값을 지정하기 위해서는 별도로 value 속성을 이용하지 않고 태그의 내부 내용을 지정한다.

cols: 초기에 설정될 열의 개수(가로 방향으로의 글자 개수)를 지정한다. 자연수여야 하며 기본값은 20.
rows: 초기에 설정될 행의 개수(세로 방향으로의 줄 수)를 지정한다. 자연수여야 하며 기본값은 2.
disabled: 비활성화 할 것인가에 대한 속성. 속성 값을 가지지 않음.
maxlength: 문자열의 최대 길이. (값이 비어있을 때 검사하지 않음).
minlength: 문자열의 최소 길이. (값이 비어있을 때 검사하지 않음.)
placeholder: 값이 비어있을 때 보여줄 텍스트.
readonly: 읽기 전용으로 지정하기 위해 사용한다. 속성 값을 가지지 않음.
required: 필수 필드이며 값이 비어있을 수 없음을 의미한다. 속성 값을 가지지 않음.
spellcheck: 브라우저에 내장되어 있는 철자 확인 기능을 사용할 것인가에 대한 설정.
	default: 기본값으로, 브라우저의 설정에 따름.
	true: 사용
	false: 미사용
